This package contains nodes for encoding/decoding camera images to/from H.\+264. At this stage, the package is still experimental and should be used with care.\hypertarget{md_nimbro_network_nimbro_cam_transport_README_autotoc_md0}{}\doxysection{Dependencies}\label{md_nimbro_network_nimbro_cam_transport_README_autotoc_md0}
The {\ttfamily sender} node uses the {\ttfamily x264} library for encoding the video stream. In most distributions, this library can be installed from the repositories (Ubuntu\+: package libx264-\/dev).

The {\ttfamily receiver} node uses the {\ttfamily ffmpeg} library for decoding. Note that the {\ttfamily avconv} framework is {\itshape not} supported. Also, most distribution packages are outdated, so it is best to compile ffmpeg from source.

Here is a minimal ffmpeg {\ttfamily configure} line that enables just the modules we need and installs {\ttfamily ffmpeg} into {\ttfamily /opt/ffmpeg}\+: \begin{DoxyVerb}./configure --enable-shared --prefix=/opt/ffmpeg --disable-everything \
  --enable-decoder=h264
\end{DoxyVerb}
