\doxysubsection{Detailed Description}
{\bfseries{This class constitutes the A\+PI of the Reflexxes Motion Libraries}} 

The class \mbox{\hyperlink{classReflexxesAPI}{Reflexxes\+A\+PI}} is the interface between the actual On-\/\+Line Trajectory Generation algorithms of the Reflexxes Motion Libraries and the applications using it. It wraps the classes of the algorithms, provides a simple and clean interface, and it hides all parts of the implementation that or not relevant the actual implementation behind its interface.~\newline
~\newline
The only two relevant functions of a {\itshape Reflexxes Motion Library} are the the methods


\begin{DoxyItemize}
\item \mbox{\hyperlink{classReflexxesAPI_a30d3cdba072553a5d53aa4ce4b4a77d1}{Reflexxes\+A\+P\+I\+::\+R\+M\+L\+Position()}}, which calls Type\+I\+V\+R\+M\+L\+Position\+::\+Get\+Next\+State\+Of\+Motion() of the class Type\+I\+V\+R\+M\+L\+Position, to execute the {\bfseries{position-\/based}} On-\/\+Line Trajectory Generation algorithm and~\newline

\item \mbox{\hyperlink{classReflexxesAPI_aaf2478f04265b6a6ac04b1b218a30678}{Reflexxes\+A\+P\+I\+::\+R\+M\+L\+Velocity()}}, which calls Type\+I\+V\+R\+M\+L\+Velocity\+::\+Get\+Next\+State\+Of\+Motion() of the class Type\+I\+V\+R\+M\+L\+Velocity, to execute the {\bfseries{velocity-\/based }} On-\/\+Line Trajectory Generation algorithm,
\end{DoxyItemize}

both of which calculate a new state of motion from any arbitrary initial state of motion.~\newline
~\newline
Both methods, require the specification of a set of input values


\begin{DoxyItemize}
\item \mbox{\hyperlink{classRMLPositionInputParameters}{R\+M\+L\+Position\+Input\+Parameters}} / \mbox{\hyperlink{classRMLVelocityInputParameters}{R\+M\+L\+Velocity\+Input\+Parameters}} (cf. \mbox{\hyperlink{page_InputValues}{Description of Input Values}})
\end{DoxyItemize}

and a data structure specifying a certain behavior of the algorithm


\begin{DoxyItemize}
\item \mbox{\hyperlink{classRMLPositionFlags}{R\+M\+L\+Position\+Flags}} / \mbox{\hyperlink{classRMLVelocityFlags}{R\+M\+L\+Velocity\+Flags}}.
\end{DoxyItemize}

Based on these input values, the result, that is, the state of motion for the succeeding control cycle, is written to an


\begin{DoxyItemize}
\item \mbox{\hyperlink{classRMLPositionOutputParameters}{R\+M\+L\+Position\+Output\+Parameters}} / \mbox{\hyperlink{classRMLVelocityOutputParameters}{R\+M\+L\+Velocity\+Output\+Parameters}} (cf. \mbox{\hyperlink{page_OutputValues}{Description of Output Values}})
\end{DoxyItemize}

object.~\newline
 To get an overview about the source code, please refer to the \mbox{\hyperlink{page_SourceCode}{Overview About the Source Code}}. A brief survey of the Reflexxes algorithm can be found at the page \mbox{\hyperlink{page_TypeIIAndIVOverview}{The Position-\/based Type II On-\/\+Line Trajectory Generation Algorithm}}, and error-\/handling procedures are described at page\+\_\+\+Error\+Handling.~\newline


Besides the classes for the input and output parameters, this class is is the only one that has to be embedded in a user application. The interface only consists of the constructor and the two methods \mbox{\hyperlink{classReflexxesAPI_a30d3cdba072553a5d53aa4ce4b4a77d1}{R\+M\+L\+Position()}} and \mbox{\hyperlink{classReflexxesAPI_aaf2478f04265b6a6ac04b1b218a30678}{R\+M\+L\+Velocity()}}, such that an integration into existing systems can be done in a clear and straight-\/forward way.

{\footnotesize\ttfamily enum \mbox{\hyperlink{classReflexxesAPI_a9bf1fa242f1f7005850cd3f517ea8f59}{Reflexxes\+A\+P\+I\+::\+R\+M\+L\+Result\+Value}}}



Result values of the On-\/\+Line Trajectory Generation algorithm. 

This enumeration contains all possible return values of the methods \mbox{\hyperlink{classReflexxesAPI_a30d3cdba072553a5d53aa4ce4b4a77d1}{Reflexxes\+A\+P\+I\+::\+R\+M\+L\+Position()}} and \mbox{\hyperlink{classReflexxesAPI_aaf2478f04265b6a6ac04b1b218a30678}{Reflexxes\+A\+P\+I\+::\+R\+M\+L\+Velocity()}}. Except \mbox{\hyperlink{classReflexxesAPI_a9bf1fa242f1f7005850cd3f517ea8f59aa85adfd03ec7dec1a88ba47c53d281ac}{Reflexxes\+A\+P\+I\+::\+R\+M\+L\+\_\+\+W\+O\+R\+K\+I\+NG}} and \mbox{\hyperlink{classReflexxesAPI_a9bf1fa242f1f7005850cd3f517ea8f59a5108326a7898a342063eb558c5614958}{Reflexxes\+A\+P\+I\+::\+R\+M\+L\+\_\+\+F\+I\+N\+A\+L\+\_\+\+S\+T\+A\+T\+E\+\_\+\+R\+E\+A\+C\+H\+ED}}, none the values must appear in a correct implementation. The input values of the algorithm are tested by the methods \mbox{\hyperlink{classRMLPositionInputParameters_a8a36aa4df4c86606a1d47f204cb901a3}{R\+M\+L\+Position\+Input\+Parameters\+::\+Check\+For\+Validity()}} and \mbox{\hyperlink{classRMLVelocityInputParameters_a73ad2f505ef37bef4a8b3b8fdbae95b2}{R\+M\+L\+Velocity\+Input\+Parameters\+::\+Check\+For\+Validity()}}.~\newline
~\newline
If either one of these methods returns {\ttfamily false}, a correct calculation of output values can not necessarily be obtained anymore, but -\/ in any case -\/ a valid and continuous motion trajectory will be generated. In such a case, the library will try to calculate correct output values, and in many cases the library is capable to provide such values successfully, but a guarantee cannot be given (cf. \mbox{\hyperlink{page_ErrorHandling}{Safety\+: Three Layers of Error Handling}}).~\newline
~\newline
If \mbox{\hyperlink{classRMLPositionInputParameters_a8a36aa4df4c86606a1d47f204cb901a3}{R\+M\+L\+Position\+Input\+Parameters\+::\+Check\+For\+Validity()}} or \mbox{\hyperlink{classRMLVelocityInputParameters_a73ad2f505ef37bef4a8b3b8fdbae95b2}{R\+M\+L\+Velocity\+Input\+Parameters\+::\+Check\+For\+Validity()}} return {\ttfamily false}, it may even be in indication for a wrong implementation of the application using the library, and it may happen that one the error-\/indicating values will be returned by \mbox{\hyperlink{classReflexxesAPI_a30d3cdba072553a5d53aa4ce4b4a77d1}{Reflexxes\+A\+P\+I\+::\+R\+M\+L\+Position()}} or \mbox{\hyperlink{classReflexxesAPI_aaf2478f04265b6a6ac04b1b218a30678}{Reflexxes\+A\+P\+I\+::\+R\+M\+L\+Velocity()}}, respectively. Furtheron, it may happen in this case that the execution time exceeds the value of R\+M\+L\+\_\+\+M\+A\+X\+\_\+\+E\+X\+E\+C\+U\+T\+I\+O\+N\+\_\+\+T\+I\+ME, such that R\+M\+L\+\_\+\+E\+R\+R\+O\+R\+\_\+\+E\+X\+E\+C\+U\+T\+I\+O\+N\+\_\+\+T\+I\+M\+E\+\_\+\+T\+O\+O\+\_\+\+B\+IG will be returned.

\begin{DoxyNote}{Note}
If \mbox{\hyperlink{classRMLPositionInputParameters_a8a36aa4df4c86606a1d47f204cb901a3}{R\+M\+L\+Position\+Input\+Parameters\+::\+Check\+For\+Validity()}} or \mbox{\hyperlink{classRMLVelocityInputParameters_a73ad2f505ef37bef4a8b3b8fdbae95b2}{R\+M\+L\+Velocity\+Input\+Parameters\+::\+Check\+For\+Validity()}}, respectively, return {\ttfamily true}, either \mbox{\hyperlink{classReflexxesAPI_a9bf1fa242f1f7005850cd3f517ea8f59aa85adfd03ec7dec1a88ba47c53d281ac}{Reflexxes\+A\+P\+I\+::\+R\+M\+L\+\_\+\+W\+O\+R\+K\+I\+NG}} or \mbox{\hyperlink{classReflexxesAPI_a9bf1fa242f1f7005850cd3f517ea8f59a5108326a7898a342063eb558c5614958}{Reflexxes\+A\+P\+I\+::\+R\+M\+L\+\_\+\+F\+I\+N\+A\+L\+\_\+\+S\+T\+A\+T\+E\+\_\+\+R\+E\+A\+C\+H\+ED}} will be returned.
\end{DoxyNote}

This is the method for velocity-\/based On-\/\+Line Trajectory Generation. 

This method executes the velocity-\/based Reflexxes On-\/\+Line Trajectory Generation algorithm.~\newline
~\newline
The implementation and the required mathematical back-\/end is much simpler than for the position-\/based method (\mbox{\hyperlink{classReflexxesAPI_a30d3cdba072553a5d53aa4ce4b4a77d1}{Reflexxes\+A\+P\+I\+::\+R\+M\+L\+Position()}}). The position-\/based algorithm is implemented in the class Type\+I\+V\+R\+M\+L\+Position, and the velocity-\/based algorithm is implemented in Type\+I\+V\+R\+M\+L\+Velocity. Basically, a call of this method behaves in the very same way than a call of \mbox{\hyperlink{classReflexxesAPI_a30d3cdba072553a5d53aa4ce4b4a77d1}{R\+M\+L\+Position()}}, but it is not possible to specify a desired target position vector $ \vec{P}_i{\,trgt} $ and a maximum velocity vector $ \vec{V}_i{\,max} $. Internally, the method Type\+I\+V\+R\+M\+L\+Velocity\+::\+Get\+Next\+State\+Of\+Motion() is called. For further details, please refer to this method.~\newline
~\newline
Based on the current state of motion $ {\bf M}_i $, the kinematic motion constraints $ {\bf B}_i $, the desired target velocity vector $ \vec{V}_i^{\,trgt} $, and the selection vector $ {\bf S}_i $ (all of which are contained in {\ttfamily Input\+Values}, a new state of motion $ {\bf M}_{i+1} $ is calculated, which can be utilized as a set-\/point value at the next control cycle. The time difference between $ {\bf M}_{i} $ and $ {\bf M}_{i+1} $ is \mbox{\hyperlink{classReflexxesAPI_aba2dd2475a01d38b8ee9c3a119ea22b9}{Reflexxes\+A\+P\+I\+::\+Cycle\+Time}} (given in seconds).
