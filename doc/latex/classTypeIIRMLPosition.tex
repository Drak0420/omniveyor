\doxysubsection{Detailed Description}
{\bfseries{This class constitutes the low-\/level user interface of the Reflexxes Type II Motion Library, which contains the Type II On-\/\+Line Trajectory Generation algorithm}} 

This class is the low-\/level application interface of the Type II On-\/\+Line Trajectory Generation algorithm. The wrapper class \mbox{\hyperlink{classReflexxesAPI}{Reflexxes\+A\+PI}}, simplifies this interface for the user, such that all relevant functionalities can be accessed, but all parts that are not needed by the user are hidden.~\newline
~\newline
The mathematical futtocks of the algorithm are described in~\newline
~\newline
{\bfseries{T. Kroeger.}}~\newline
{\bfseries{On-\/\+Line Trajectory Generation in Robotic Systems.}}~\newline
{\bfseries{Springer Tracts in Advanced Robotics, Vol. 58, Springer, January 2010.}}~\newline
{\bfseries{\href{http://www.springer.com/978-3-642-05174-6}{\texttt{ http\+://www.\+springer.\+com/978-\/3-\/642-\/05174-\/6}}}}~\newline
~\newline
The algorithm makes use of the namespace Type\+I\+I\+R\+M\+L\+Math, which is a collection of mathematical functions required for the algorithm. Besides others, this namespace contains the functions for each single decision of all decision trees as well as all functions to solve the systems of equations that occur in the leaves of of the decision trees.~\newline
 
\begin{DoxyItemize}
\item The decision trees itself are part of this class. 
\item Furthermore, all {\itshape management} functionalities (e.\+g., the usage of time-\/ {\itshape or} phase-\/synchronization criteria, cf. \mbox{\hyperlink{page_SynchronizationBehavior}{Synchronization Behavior}}), 
\item the three-\/layered error handling mechanism to ensure valid output values in {\itshape any} case, 
\item verification and scalings of input and output values, 
\item the implementation of 
\begin{DoxyItemize}
\item {\bfseries{Step 1}}, that is, calculating $ t_i^{\,sync} $ by using the the decision trees 1A, 1B, and 1C~\newline
 
\item {\bfseries{Step 2}}, that is, calculating all parameters $ {\cal M}_{i}(t) $ of the synchronized motion trajectory~\newline
 
\item {\bfseries{Step 3}}, that is, calculating the output values of the algorithm~\newline
 
\end{DoxyItemize}are part of this class, and 
\item the calculation of extremum states of motion is done by this class. 
\end{DoxyItemize}

Besides the constructor, the class only has one method that is relevant for the user\+: \mbox{\hyperlink{classTypeIIRMLPosition_af2a8f0d8c55c3f5a4861149a7bc35d66}{Type\+I\+I\+R\+M\+L\+Position\+::\+Get\+Next\+State\+Of\+Motion()}}. The input and and output values of this function are described in the classes \mbox{\hyperlink{classRMLFlags}{R\+M\+L\+Flags}}, \mbox{\hyperlink{classRMLPositionInputParameters}{R\+M\+L\+Position\+Input\+Parameters}}, and \mbox{\hyperlink{classRMLPositionOutputParameters}{R\+M\+L\+Position\+Output\+Parameters}}, all of which are also described in the context of the class \mbox{\hyperlink{classReflexxesAPI}{Reflexxes\+A\+PI}}. Information about all these parameters can be found on the pages


\begin{DoxyItemize}
\item \mbox{\hyperlink{page_TypeIIAndIVOverview}{The Position-\/based Type II On-\/\+Line Trajectory Generation Algorithm}},
\item \mbox{\hyperlink{page_InputValues}{Description of Input Values}}, and
\item page\+\_\+\+Output\+Values.~\newline

\end{DoxyItemize}

The class \mbox{\hyperlink{classTypeIIRMLVelocity}{Type\+I\+I\+R\+M\+L\+Velocity}} is a byproduct only. It can be used in the very same as this class, but target position vectors $ \vec{P}_i^{\,trgt} $ cannot be specified. While the class \mbox{\hyperlink{classTypeIIRMLPosition}{Type\+I\+I\+R\+M\+L\+Position}} can be used for the instantaneous computation of motions towards a desired target pose or position, the class \mbox{\hyperlink{classTypeIIRMLVelocity}{Type\+I\+I\+R\+M\+L\+Velocity}} basically has two functionalities\+:


\begin{DoxyEnumerate}
\item \mbox{\hyperlink{classTypeIIRMLVelocity}{Type\+I\+I\+R\+M\+L\+Velocity}} is the main component for the {\itshape second} safety layer (cf. \mbox{\hyperlink{page_ErrorHandling}{Safety\+: Three Layers of Error Handling}}).
\item \mbox{\hyperlink{classTypeIIRMLVelocity}{Type\+I\+I\+R\+M\+L\+Velocity}} can be used to generate a trajectory starting from an arbitrary initial state of motion to achieve a certain target velocity $ \vec{V}_i^{\,trgt} $.
\end{DoxyEnumerate}

{\footnotesize\ttfamily void Type\+I\+I\+R\+M\+L\+Position\+::\+Compare\+Initial\+And\+Target\+Stateof\+Motion (\begin{DoxyParamCaption}\item[{void}]{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [protected]}}



If the initial state of motion {\itshape exactly} equals the target state of motion, an adaptation is performed. 

If the initial state of motion {\itshape exactly} equals the target state of motion, we add a negligible error to the input state of motion in order to let the decision trees run deterministically. Otherwise, these values would be a singularity for the decision trees as


\begin{DoxyItemize}
\item either {\itshape no trajectory} is required (i.\+e., an execution time of zero seconds), or
\item the current motion is continued until the same state of motion is reached again (time-\/optimally).
\end{DoxyItemize}

As this case can only occur, if the input values change, {\itshape and} if the output values of the last cycle are not directly fed back to the input parameters of this cycle, we need to calculate the trajectory to reach the desired state of motion. Otherwise, no trajectory would be required at all and it would not make sense to the On-\/\+Line Trajectory Generation algorithm.

In case of an error, this method triggers the second layer of the safety concept. 

If no trajectory can be calculated by the position-\/based On-\/\+Line Trajectory Generation algorithm (\mbox{\hyperlink{classTypeIIRMLPosition_af2a8f0d8c55c3f5a4861149a7bc35d66}{Type\+I\+I\+R\+M\+L\+Position\+::\+Get\+Next\+State\+Of\+Motion()}}), the velocity-\/based algorithm (\mbox{\hyperlink{classTypeIIRMLVelocity_aa9725881ff7c9355cfd1c4446cff0ffc}{Type\+I\+I\+R\+M\+L\+Velocity\+::\+Get\+Next\+State\+Of\+Motion()}}) is called in the second safety layer. Before this call can be made, this method casts the \mbox{\hyperlink{classTypeIIRMLPosition_a221c7609d4f5c15f75f6622cccdcc328}{Type\+I\+I\+R\+M\+L\+Position\+::\+Current\+Input\+Parameters}} object and the \mbox{\hyperlink{classRMLPositionFlags}{R\+M\+L\+Position\+Flags}} input flag object used in \mbox{\hyperlink{classTypeIIRMLPosition_af2a8f0d8c55c3f5a4861149a7bc35d66}{Type\+I\+I\+R\+M\+L\+Position\+::\+Get\+Next\+State\+Of\+Motion()}} to an \mbox{\hyperlink{classRMLVelocityInputParameters}{R\+M\+L\+Velocity\+Input\+Parameters}} object and an \mbox{\hyperlink{classRMLVelocityFlags}{R\+M\+L\+Velocity\+Flags}} object. During this casting, the desired target velocity vector $ \vec{V}_{i}^{\,trgt} $ for the velocity-\/based On-\/\+Line Trajectory Generation algorithm is either set


\begin{DoxyItemize}
\item to \mbox{\hyperlink{classRMLPositionInputParameters_a04d8c3135843bae93cbf7b8cbab8d819}{R\+M\+L\+Position\+Input\+Parameters\+::\+Alternative\+Target\+Velocity\+Vector}} of the \mbox{\hyperlink{classTypeIIRMLPosition_a221c7609d4f5c15f75f6622cccdcc328}{Type\+I\+I\+R\+M\+L\+Position\+::\+Current\+Input\+Parameters}} object if the flag \mbox{\hyperlink{classRMLPositionFlags_aeb8be0d5bdaffa0384f2c9915c5134a1}{R\+M\+L\+Position\+Flags\+::\+Keep\+Current\+Velocity\+In\+Case\+Of\+Fallback\+Strategy}} is set to {\ttfamily false} or
\item to the current velocity vector $ \vec{V}_{i} $ if the flag \mbox{\hyperlink{classRMLPositionFlags_aeb8be0d5bdaffa0384f2c9915c5134a1}{R\+M\+L\+Position\+Flags\+::\+Keep\+Current\+Velocity\+In\+Case\+Of\+Fallback\+Strategy}} is set to {\ttfamily true}.
\end{DoxyItemize}

Subsequently to the casting procedure, \mbox{\hyperlink{classTypeIIRMLVelocity_aa9725881ff7c9355cfd1c4446cff0ffc}{Type\+I\+I\+R\+M\+L\+Velocity\+::\+Get\+Next\+State\+Of\+Motion()}} is called and generates valid and feasible output values, which are represented in a \mbox{\hyperlink{classRMLVelocityOutputParameters}{R\+M\+L\+Velocity\+Output\+Parameters}} that finally is casted to a \mbox{\hyperlink{classRMLPositionOutputParameters}{R\+M\+L\+Position\+Output\+Parameters}} object, namely {\ttfamily Output\+Values}.

This method modifies the selection vector \mbox{\hyperlink{classRMLInputParameters_a2d0492e641eb7018e39011635d6135b2}{R\+M\+L\+Position\+Input\+Parameters\+::\+Selection\+Vector}} of \mbox{\hyperlink{classTypeIIRMLPosition_a221c7609d4f5c15f75f6622cccdcc328}{Type\+I\+I\+R\+M\+L\+Position\+::\+Current\+Input\+Parameters}} to \mbox{\hyperlink{classTypeIIRMLPosition_ab57be7385519fef99da73468230dae9e}{Type\+I\+I\+R\+M\+L\+Position\+::\+Modified\+Selection\+Vector}}. The trajectory parameters for all other degrees of freedom $ \left\{1,\,\dots,\,K\right\}\backslash\left\{\kappa\right\} $ are calculated using the reference vector $ \vec{\varrho}_i $ \mbox{\hyperlink{classTypeIIRMLPosition_a1ba25eade6d0fdb1f4c522e2023e96c4}{Type\+I\+I\+R\+M\+L\+Position\+::\+Phase\+Synchronization\+Reference\+Vector}}.~\newline
~\newline
In order to compensate numerical inaccuracies, the resulting polynomials for the degrees of freedom $ \left\{1,\,\dots,\,K\right\}\backslash\left\{\kappa\right\} $ are adapted. Therefore, a first-\/order polynomial is added to the polynomials represented by \mbox{\hyperlink{classTypeIIRMLPosition_af5581fa0d1116939907c15e446331610}{Type\+I\+I\+R\+M\+L\+Position\+::\+Polynomials}}.
